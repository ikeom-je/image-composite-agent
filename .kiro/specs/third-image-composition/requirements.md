# 第3画像合成機能 - 要件定義

## 概要

既存の画像合成REST APIシステムに第3の画像を合成する機能を追加する。現在の2画像合成システム（image1, image2）を拡張し、image3パラメータを追加して3つの画像を同時に合成できるようにする。また、テスト用の三角形画像を生成・追加し、フロントエンドでも3つ目の画像を選択できるようにする。

## 要件

### Requirement 1

**User Story:** ユーザーとして、3つの画像を指定して合成画像を生成したい。そうすることで、より複雑で豊かな画像コンテンツを作成できる。

#### Acceptance Criteria

1. WHEN ユーザーが3つの画像パラメータ（image1, image2, image3）を指定する THEN システムは3画像合成処理を実行する
2. WHEN 画像合成処理が完了する THEN 3つの画像が合成されたPNG形式の画像が生成される
3. WHEN image3パラメータが指定されない THEN 従来通り2画像合成処理を実行する（後方互換性）
4. WHEN 3つの画像にアルファチャンネル（透過情報）がある THEN 透過情報を保持して合成される
5. WHEN 3つの画像が重なる場合 THEN image1 → image2 → image3の順序で前面に表示される

### Requirement 2

**User Story:** ユーザーとして、第3画像の位置とサイズを自由に調整したい。そうすることで、理想的なレイアウトの3画像合成を作成できる。

#### Acceptance Criteria

1. WHEN 第3画像の位置パラメータ（image3X, image3Y）を指定する THEN 指定した位置に画像が配置される
2. WHEN 第3画像のサイズパラメータ（image3Width, image3Height）を指定する THEN 指定したサイズに画像がリサイズされる
3. WHEN 第3画像の位置パラメータが指定されない THEN デフォルトの位置（左上）に画像が配置される
4. WHEN 第3画像のサイズパラメータが指定されない THEN デフォルトのサイズ（300x200）で画像が表示される
5. WHEN 3つの画像が重なる場合 THEN 適切な重ね順で表示される

### Requirement 3

**User Story:** 開発者として、テスト用の三角形画像を生成・追加したい。そうすることで、3画像合成機能のテストと動作確認ができる。

#### Acceptance Criteria

1. WHEN "triangle_green.png"という三角形画像を生成する THEN 緑色の三角形画像がPNG形式で作成される
2. WHEN 三角形画像を生成する THEN アルファチャンネル（透過背景）を持つ画像として作成される
3. WHEN 三角形画像をS3にアップロードする THEN lambda/python/imagesディレクトリと同じ場所に保存される
4. WHEN image3="test"を指定する THEN 三角形画像が自動的に選択される
5. WHEN 既存のテスト画像と組み合わせる THEN 円・四角・三角の3つの図形が合成される

### Requirement 4

**User Story:** ユーザーとして、フロントエンドで3つ目の画像を選択したい。そうすることで、直感的なWebインターフェースで3画像合成を行える。

#### Acceptance Criteria

1. WHEN フロントエンドにアクセスする THEN 第3画像選択のフォーム要素が表示される
2. WHEN 第3画像のプルダウンメニューを操作する THEN テスト画像とS3画像を選択できる
3. WHEN 第3画像の位置・サイズ調整フォームを操作する THEN 数値入力で配置を調整できる
4. WHEN 3画像合成を実行する THEN API経由で3つの画像が合成される
5. WHEN 使用例を選択する THEN 3画像合成のプリセットパラメータで実行される

### Requirement 5

**User Story:** 開発者として、既存システムとの互換性を保持したい。そうすることで、既存のAPIユーザーに影響を与えずに機能拡張できる。

#### Acceptance Criteria

1. WHEN image3パラメータが指定されない THEN 従来通りの2画像合成処理を実行する
2. WHEN 既存のAPIエンドポイントを呼び出す THEN 既存の動作が変更されない
3. WHEN 既存のフロントエンド機能を使用する THEN 従来通りの操作ができる
4. WHEN 既存のテスト画像を使用する THEN 正常に動作する
5. WHEN HTMLレスポンスを表示する THEN 3画像対応の情報が表示される

### Requirement 6

**User Story:** 開発者として、3画像合成機能のテストを実装したい。そうすることで、機能の品質と信頼性を確保できる。

#### Acceptance Criteria

1. WHEN Lambda関数のユニットテストを実行する THEN 3画像合成処理の正確性を検証する
2. WHEN API E2Eテストを実行する THEN 3画像合成エンドポイントの動作を検証する
3. WHEN フロントエンドE2Eテストを実行する THEN 3画像選択UI機能の動作を検証する
4. WHEN 後方互換性テストを実行する THEN 既存の2画像合成機能が正常動作することを確認する
5. WHEN 三角形画像テストを実行する THEN 新しいテスト画像が正常に使用できることを確認する

## 技術的制約

- 既存のAWS CDK + Lambda + API Gateway + S3構成を維持
- Python 3.12 + Pillowライブラリによる画像処理を継続
- Vue.js 3 + Vite + Tailwind CSSフロントエンドとの統合
- 既存のAPIエンドポイント（/images/composite）を拡張
- 後方互換性の完全な保持
- アルファチャンネル（透過情報）の完全サポート

## 成功基準

1. 3つの画像の高品質な合成処理
2. 既存システムとの完全な後方互換性
3. 新しい三角形テスト画像の正常動作
4. フロントエンドでの3画像選択機能
5. 包括的なテストカバレッジ
6. 既存のパフォーマンスレベルの維持
7. アルファチャンネル対応の継続
