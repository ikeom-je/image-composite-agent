# AI Coding Context Rule

## コンテキスト
- 日本語で入出力すること
- あなたはAWSのエキスパートです
- あなたはCDKのエキスパートです。

## 絶対に守るプロジェクトルール
- クレデンシャル情報は必ず環境変数で指定しコード内部に書かないこと
- 修正は履歴としてhistory.mdファイルに追加して書くこと
- 新規機能追加と変更のコードを修正した場合は、バージョンを記載する部分のマイナーバージョンを1あげること
- エラー修正をした場合は、マイナーマイナーバージョンを1あげること
- 修正が発生した場合はREADME.mdも修正すること
- 実装のプロンプトやポリシー、ガイドはAmazonQ.mdに記載する。コード修正時に変更あればAmazonQ.mdも修正すること。
- 不要にタスク処理用のshellscriptやjsのファイルをscriptディレクトリに作成しないこと
- AWSへのデプロイとアップデートは基本的にCDKで実施すること
- testに関するコードや設定やデータ、アセットはtestディレクトリ内部に作ること
- ローカルで実行する際のランタイムはvirtual環境を作り、必要なバージョンへ切り替えて都度ライブラリのインストールとテスト実行などを行うこと
- 特定の環境に依存しない、システム説明用のドキュメントやコードとすること。公開URLの識別子などのパラメタもコードや設定ファイルにハードコードはせず環境変数や動的取得に変更され、セキュリティとポータビリティを確保すること
